# Contributing

This repository holds public documents that stay open to revision rather than being frozen at a version of record. See `ai-research-workflow/ai-research-workflow.md` for the full rationale behind this store and the workflow it is part of.

## How to contribute

- Open a pull request. Direct pushes to `main` are disabled; every change, including the author's own, goes through review.
- Small fixes (typos, broken links, formatting) can go straight to a PR with a short description.
- Substantive changes (new sections, revised arguments, added claims) should reference the discussion or reasoning behind the change in the PR description, so the revision history stays legible.
- Keep the document's own `Document Metadata` block current (version, last-updated date, branch, revision note) — see the metadata convention described in the document itself.

## Citations and authorship

- If your contribution adds a citation, verify it resolves to a real source before submitting — do not rely solely on an AI assistant's suggested reference.
- Substantial written contributions may be credited in the document (e.g., an acknowledgments note or co-author line) at the maintainer's discretion; open an issue first if you're proposing a large addition, so authorship expectations are clear before you invest the work.
